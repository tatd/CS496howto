Finding Data

For a full list of what objects each API request returns, you can view the official documentation. Pokeapi contains a lot of video game data that we won't need for a basic Pokedex, so I will show you some sample requests you can make to gather the relevant information.

As a reminder, here is the request that we are still working with:

var req = new XMLHttpRequest();
var URL = "http://pokeapi.co/api/v1/pokemon/25/";

req.open("GET", URL, true);

req.addEventListener("load", function(){
    var response = JSON.parse(req.responseText);
    //var response2 = JSON.stringify(response, null, "  ");

    //console.log(response);
    //console.log(response2);
});

req.send(null);

I've commented out the lines of code we do not need at this moment because we do not need to view the entire JSON response right now.

Some items that would be useful for a basic Pokedex would be a Pokemon's number, name, type, a description, and an image.

Number and Name

If you inspect the entire JSON object on the second page, towards the bottom, you'll see this:

...
"name": "Pikachu",
"national_id": 25,
...

The key national_id holds the value 25. This means that Pikachu is #25. In our callback function, we set var response = JSON.parse(req.responseText); So we can access the number as an object of response, so it would look like response.objectName. In this case, we would use response.national_id.

Similarly, the key name holds the value "Pikachu". So the name of the Pokemon in question is Pikachu, which is what we expected. This can be accessed with response.name.

Type

Also towards the bottom of the JSON object is an array named types:

...
  "types": [
    {
      "name": "electric",
      "resource_uri": "/api/v1/type/13/"
    }
  ],
...

This is an array because some Pokemon have up to two different types. In this case, Pikachu just has one type, which is electric. We can see the first object in this array has a key of name with value "electric". The resource_uri contains a value that we can make an API request to in order to get data about the electric type.

To access the "electric" string value, we must use array index notation, which would be response.objectName[index].key. So in this case, we would use response.types[0].name. The first element in an array is index 0. If Pikachu had two types, then the second type name would be accessed by response.types[1].name, because the second element of an array has index of 1.

Viewing the Desired Data

Now that we have the code to access the data we want for our Pokedex, we want to display that data. An easy way to do this is to log the data to the console. To do this, we can add log functions to our load event, logging the data mentioned above: 

var req = new XMLHttpRequest();
var URL = "http://pokeapi.co/api/v1/pokemon/25/";

req.open("GET", URL, true);

req.addEventListener("load", function(){
    var response = JSON.parse(req.responseText);

/*********** New Stuff ****************************/
    console.log("Number: ", response.national_id);
    console.log("Name: ", response.name);
    console.log("Type: ", response.types[0].name);
/**************************************************/
    
    //var response2 = JSON.stringify(response, null, "  ");

    //console.log(response);
    //console.log(response2);
});

req.send(null);

Here is the console log after making the above request: 

Number: 25
Name: Pikachu
Type: electric

We can see here that the above function calls returned the appropriate data.

This data can be displayed within a linked HTML document. For example, if you wanted to display the number 25, in the HTML document, you can define a <span id="number"></span>. Then, in your linked JavaScript document, you could display the number with document.getElementById("number").textContent = response.national_id;.

Pokeapi Guide

with jQuery Mobile and PhoneGap/Cordova

By Dennis Tat


This is a guide on how to make a simple single-page mobile app with Pokeapi in conjunction with jQuery Mobile and PhoneGap/Cordova. Pokeapi is a web API that contains data on the Pokemon video games. From Pokeapi, you can get information on over 700 Pokemon such as their names, descriptions, types, and even images of the sprites used in the video games. Using this information, you could build your own web-based Pokedex (which is basically an encyclopedia of Pokemon). By using jQuery Mobile and Phonegap, this application can be easily be made into a cross-platform, native mobile application.

Pokeapi accepts HTTP requests via GET. This API does not require an API key, so anyone can immediately jump in and start making requests. However, there is a daily limit of 300 requests for each resource (so each specific URL can be called up to 300 times per day).

This guide will walk through how to start making requests to Pokeapi using the previously mentioned tools and go through some sample calls so that you can start constructing your own Pokedex.

We'll start with building the app with just jQuery Mobile so we can develop more quickly and view it in the web browser. Then we'll wrap the project with Phonegap to make it a native application. You can find all the documentation on how to start projects with jQuery Mobile and Phonegap at the links mentioned above.

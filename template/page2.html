That is a lot of data. And that was only from one request. If you look back at the URL, the Pokemon data we requested was for Pokemon #25. There were 151 Pokemon in the first generation of Pokemon. This franchise now spans 20 years, or six "generations." So that means for Pokemon #25, a Pokemon from Generation I, there is 20 years worth of data. For a newer Pokemon from Generation VI, there would be less data because that Pokemon has been in fewer games.

From quickly scanning the JSON object, or if you're familiar with Pokemon, you can tell that Pokemon #25 is Pikachu. If you couldn't tell, that's OK. We will go over how to extract the Pokemon's name from the JSON object later.

Reading the Response

To break things down, let's look at the first item in our response:

{
  "abilities": [
    {
      "name": "static",
      "resource_uri": "/api/v1/ability/9/"
    },
    {
      "name": "lightningrod",
      "resource_uri": "/api/v1/ability/31/"
    }
  ],
...

Our first key is abilities, which is actually an array. This array holds the data of Pikachu's abilities. Each item in the array represents a different ability.

The first item in the array has the key name with the value "static". This means Pikachu's first ability is named "static." The next key for the first item in the array is a resource_uri to the value "/api/v1/ability/9/", which is another URL we can make a request to to get data about the "static" ability.

Similarly, the second item in the array is Pikachu's second ability. The second ability has the "name": "lighteningrod" and also has a resource_uri for that ability.
